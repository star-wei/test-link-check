# 门窗装箱方案系统 - 需求文档

## 1. 项目背景

### 1.1 业务场景
- 报价单由人工分割到各工厂
- 工厂内部需要为订单生成装箱方案
- 部分产品厂家自包装，部分由系统处理

### 1.2 系统边界
- **输入**：订单编号 + 产品编号（通过API获取数据）
- **输出**：装箱方案（箱型 + 每箱装什么 + 怎么摆放）
- **范围**：工厂内部的装箱环节

---

## 2. 产品分类

| 类型 | 处理方式 | 说明 |
|------|----------|------|
| 入户门 | 厂家自包装 | 不参与系统计算 |
| 室内门 | 厂家自包装 | 不参与系统计算 |
| 车库门 | 厂家自包装 | 不参与系统计算 |
| 折叠门 | 厂家自包装 | 不参与系统计算 |
| **窗等其他产品** | **系统处理** | 需要生成装箱方案 |

---

## 3. 数据输入

### 3.1 API 数据源
- 通过订单编号 + 产品编号调用API
- 获取产品原始尺寸、数量、扇数等信息

### 3.2 本地基础数据（Excel）
用于补充包装参数和产品重量：

| 字段 | 类型 | 说明 |
|------|------|------|
| 厂家 | string | 产品所属厂家 |
| 产品编号 | string | 唯一标识 |
| 产品类型 | enum | window/door/other |
| 原始宽度 | number | mm |
| 原始高度 | number | mm |
| 原始厚度 | number | mm |
| 包装后宽度 | number | mm |
| 包装后高度 | number | mm |
| 包装后厚度 | number | mm |
| 单扇重量 | number | kg |

---

## 4. 装箱约束条件

### 4.1 箱体规格限制

| 维度 | 范围 | 说明 |
|------|------|------|
| 宽度 | 500-1200mm | 可调范围 |
| 高度 | ≤ 2500mm | 最大限制 |
| 长度 | 基本不限 | 根据产品高度 |
| 总重量 | ≤ 1500kg | 单箱最大承重 |

### 4.2 摆放规则

```
箱子三维度分配：
┌─────────────────────────────────────┐
│  箱子长度方向  →  窗子高度方向       │
│  箱子高度方向  →  窗子宽度方向       │
│  箱子宽度方向  →  窗子厚度叠放方向   │
└─────────────────────────────────────┘
```

**关键点：**
- 窗子**竖放**（高度沿箱子长度）
- **单扇宽度**必须在 500-1200mm 范围内
- **厚度方向可叠放**多扇，累计不超过 2500mm
- **单箱总重量**不超过 1500kg

## 6. 功能模块

### 6.1 数据导入模块
- **订单查询**：输入订单编号，调用API获取产品列表
- **基础数据维护**：上传/更新Excel（厂家-产品-包装参数映射）
- **数据校验**：检查API返回数据与本地基础数据的匹配

### 6.2 装箱计算模块
- **自动计算**：根据约束条件生成初步装箱方案
- **异常标记**：自动识别需要人工干预的订单
- **算法优化**：尽量提高空间利用率，减少箱子数量

### 6.3 可视化调整模块
- **方案展示**：显示每箱装什么、各扇尺寸、利用率
- **手动调整**：
  - 修改单扇实际尺寸（应对不均匀分割）
  - 调整分箱（哪些扇放同一箱）
  - 修改箱子规格
- **重新计算**：调整后一键重新生成方案

### 6.4 方案导出模块
- **装箱清单**：PDF格式，包含每箱明细
- **标签打印**：箱号标签、产品标签
- **数据导出**：Excel格式，对接下游系统

## 7. 特殊场景处理

### 7.1 不均匀分割窗
**场景**：中间宽、左右窄的三扇窗

**处理**：
1. 系统初装按平均估算（总宽÷扇数）
2. 标记"需人工确认"
3. 人工界面显示原始总宽和扇数
4. 操作员输入实际每扇宽度
5. 重新计算装箱方案

### 7.2 超重处理
**场景**：单扇重量过大，无法与其他扇同箱

**处理**：
1. 系统计算每箱累计重量
2. 接近1500kg时自动分箱
3. 超重产品强制单独装箱

### 7.3 尺寸超限
**场景**：单扇宽度 > 1200mm

**处理**：
1. 系统标记"必须拆分"
2. 人工确认拆分方式
3. 支持按扇数拆或不拆（特殊包装）

## 9. 非功能需求

### 9.1 性能
- 单订单计算响应时间 < 3秒
- 支持同时处理10个订单

### 9.2 可靠性
- 调整操作支持撤销/重做
- 方案生成后自动保存草稿

### 9.3 易用性
- 异常订单明显标记
- 操作有明确反馈
- 支持键盘快捷键
