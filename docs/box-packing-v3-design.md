## 二、混合拆件策略（方案 C）

### 2.1 自动拆件规则库

| 产品类型 | 自动拆件规则 | 备注 |
|---|---|---|
| **固定窗（Fixed Window）** | 宽度按 `1143mm` 等分扇数，向上取整 | 边扇可能加 30mm 框架余量 |
| **推拉门（Sliding Door）** | 按轨位拆分，默认每扇 ≤ 1300mm | 超过则自动加扇数 |
| **入户门（Entry Door）** | **不拆**，整扇作为一个组件 | 含门框包装 |
| **车库门（Garage Door）** | 按门板单元拆分，每扇宽度固定 | 参考标准门板尺寸 |
| **折叠门（Folding Door）** | 按折叠扇数量拆分 | 从图纸参数或规格字段读取 |
| **铝木/幕墙等定制产品** | **触发人工拆件流程** | 系统无法自动拆分 |

### 2.2 人工拆件录入

对于复杂/定制产品，人工在系统中录入：
- `原始订单号`（如 DW2025-1245）
- `组件编号`（如 W1-P1, W1-P2）
- `成品尺寸`（宽 × 高 × 厚，mm）
- `玻璃类型`（双玻 / 三玻）
- `数量`

系统接收后，直接进入后续的 **Padding + 重量计算 + 装箱** 流程。

## 三、Padding 规则

**统一公式**：
- `包装宽 = 成品宽 + 60mm`
- `包装高 = 成品高 + 60mm`
- `包装厚 = 成品厚 + 30mm`

> 未来可扩展为按产品类型/易碎度动态调整 padding。

---

## 四、重量计算

**公式**：
```
组件重量(kg) = (成品宽 × 成品高 / 1,000,000) × 玻璃系数
```

| 玻璃类型 | 系数 |
|---|---|
| 双玻（Double Glazing） | 50 kg/㎡ |
| 三玻（Triple Glazing） | 75 kg/㎡ |

**示例**：
- W1-P1: 1143mm × 3048mm = 3.483 ㎡ × 75 kg/㎡ = **261.2 kg**

## 五、批量装箱优化算法

### 5.1 输入

按工厂聚合后的组件池：
```python
factory_pool = {
  "凯研工厂": [W1-P1, W1-P2, W2-P1, W2-P2, ...],
  "绿盾工厂": [D5-P1, D5-P2, ...]
}
```

### 5.2 硬约束

1. **尺寸约束**：每个组件的包装尺寸必须能单独放入箱子（6 种旋转）
2. **重量约束**：单个箱子总重量 **≤ 1500 kg**
3. **体积约束**：所有组件总体积 ≤ 箱子体积 × 0.95
4. **工厂箱型约束**：优先使用当前工厂的专属箱型；若无法满足，允许扩大到全局箱型

### 5.3 优化目标（多目标评分函数）

采用**综合评分**排序，目标之间不冲突：

| 目标 | 评分项 | 说明 |
|---|---|---|
| **A. 箱数最少** | 基础奖励 | 每箱固定加分，鼓励合并 |
| **B. 体积利用率高** | `util_score = 100 - abs(utilization - 80)` | 理想区间 70%~95% |
| **C. 重量接近 1500kg** | `weight_score = (total_weight / 1500) * 50` | 越接近 1500 分越高（但不超过 1500） |
| **D. 同系列不散装** | `group_bonus` | 同一 product_id 的组件尽量在同一箱内 |

**综合评分公式**：
```python
score = util_score + weight_score + group_bonus - volume_penalty - margin_penalty
```

### 5.4 算法流程（V3）

```
1. 为每个组件计算可行箱型 Top-K
2. 按产品系列分组，同系列组件优先尝试合并
3. 对剩余跨系列组件，按体积从大到小排序
4. 依次尝试放入已有箱子（检查尺寸+重量+体积）
5. 若无法放入任何已有箱子，则开启新箱（选择最佳匹配箱型）
6. 最终输出各工厂的装箱明细
```
