## 二、算法设计

### 2.1 核心思路
给定一个产品的三维尺寸（长×宽×高），系统会自动：
1. **枚举所有旋转方式**（6 种排列组合），判断产品是否能放入箱子。
2. **考虑防震余量**（padding），确保产品各边与箱壁之间留有一定缓冲空间。
3. **评估体积利用率**，优先推荐大小适中、利用率高的箱子。
4. **检查重量限制**，确保产品重量不超过箱子承重能力。
5. **综合评分排序**，返回 Top-K 推荐结果。

### 2.2 匹配规则
- **可旋转**：产品三边可以任意旋转，只要存在一种旋转方式能放入箱子即可。
- **余量约束**：默认整体余量 `padding = 20mm`，即产品放入后四周至少有 20mm 缓冲。
- **最小边距约束**：默认每边最小余量 `10mm`。
- **重量约束**：默认产品重量不得超过箱子净重的 `90%`（安全系数）。

### 2.3 评分逻辑
综合评分由以下因素构成：

| 因素 | 说明 |
|---|---|
| 体积利用率 | 理想区间为 70%~95%，越接近 80% 得分越高 |
| 箱子体积惩罚 | 避免推荐过大的箱子，箱子体积越大惩罚越高 |
| 余量均匀度 | 三边余量越均匀（标准差越小），得分越高 |
| 最小边距惩罚 | 若某边余量小于阈值，会大幅扣分 |
| 品牌偏好加分 | 可配置优先推荐某些品牌/类别 |

## 三、使用方法

### 3.1 运行示例
```bash
python box_packing_solver.py
```

### 3.2 作为模块调用
```python
from box_packing_solver import Product, recommend_box, BOX_DATABASE

# 定义产品
product = Product(name="铝合金门", length=2500, width=450, height=1150, weight=300)

# 获取 Top-3 推荐
results = recommend_box(product, top_k=3)

for r in results:
    print(f"箱号: {r.box.box_id}, 利用率: {r.volume_utilization:.1f}%, 评分: {r.score:.2f}")
```

### 3.3 自定义参数
```python
results = recommend_box(
    product,
    top_k=3,
    padding_mm=30,              # 增大防震余量到 30mm
    min_margin_per_side=15,     # 每边最小余量 15mm
    weight_safety_factor=0.8,   # 重量安全系数 80%
    preferred_brands=["凯研"]   # 优先推荐凯研品牌
)
```

### 3.4 批量推荐
```python
from box_packing_solver import batch_recommend

products = [
    Product("门A", 2500, 450, 1150, weight=300),
    Product("门B", 3600, 550, 750, weight=250),
]

batch_results = batch_recommend(products, top_k=1)
```

### 3.5 整体装箱方案分析（多组件订单）
```python
from box_packing_solver import analyze_packing_plan, WINDOW_4591X3067_COMPONENTS

plan = analyze_packing_plan(WINDOW_4591X3067_COMPONENTS)
print(f"总箱数: {plan['total_boxes_needed']}")
print(f"无法装箱: {plan['unfit_products']}")
```

## 四、4591×3067 门窗产品装箱案例分析

### 4.1 产品组件提取
从产品图 `product_dimensions_4591x3067.png` 中提取的尺寸：

| 组件 | 估算尺寸 (mm) | 重量 (kg) | 装箱策略 |
|---|---|---|---|
| **整扇门窗** | 4572 × 3048 × 150 | 800 | ❌ 不可直接装箱 |
| **窗扇 P1-P4** | 1100 × 100 × 2400 | 120/个 | ✅ 分拆后单独装箱 |
| **下部横梁-整体** | 4572 × 610 × 100 | 180 | ⚠️ 长度超限，建议拆分为两段 |
| **下部横梁-左半** | 2286 × 610 × 100 | 90 | ✅ 可装箱 |
| **下部横梁-右半** | 2286 × 610 × 100 | 90 | ✅ 可装箱 |
| **竖框型材** | 3048 × 80 × 60 | 45/根 | ✅ 长条型材装箱 |
| **中梃型材** | 3048 × 60 × 60 | 35/根 | ✅ 长条型材装箱 |

### 4.2 实际推荐结果
运行程序后，该门窗产品的推荐箱型如下：

| 组件 | 推荐箱型 | 品牌 | 体积利用率 | 说明 |
|---|---|---|---|---|
| 窗扇 P1-P4 (×4) | **C1** | 极筑 | ~16.5% | 2400mm 高度刚好放入 C1 的 2600mm 长边 |
| 下部横梁-整体 | **C27** | 凯研 | ~8.9% | 4980mm 长度可装 4572mm 横梁 |
| 下部横梁-左/右半 (×2) | **C19** | 凯研 | ~3.4% | 3210mm 长度足够装 2286mm |
| 竖框/中梃型材 (×5) | **C13** | 折叠门 | ~0.6-0.8% | 3760mm 长度装 3048mm 型材 |

**预计总箱数**：12 箱（4 个窗扇 + 1 个整体横梁 或 2 个拆分横梁 + 5 根型材）

> 注：若下部横梁不拆分，可用 **C27** 一箱装下；若拆分则为 **C19 × 2**。从体积利用率看，整体装箱（C27，8.9%）优于拆分（C19 × 2，各 3.4%），但需评估实际运输和搬运可行性。

### 4.3 关键发现
- **整扇门窗 4572×3048mm 远超所有箱型上限**，必须拆分运输。
- **窗扇高度 2400mm 是瓶颈**：C1 (2600mm) 是唯一能在长度方向容纳 2400mm + 余量的中小型箱子。C30 虽更大，但算法优先推荐体积更小的 C1（节省运费）。
- **长条形型材**（3048mm）需要长度 ≥ 3760mm 的箱子，C13/C14 是较合适的选择。

## 五、分级选择策略

针对不同尺寸级别的产品，建议采用以下分级策略：

### 5.1 小型配件（< 1500mm）
- **代表**：五金件、密封条、小玻璃
- **策略**：优先选择 **C1、C10、C15、C17、C21-C23** 等中小型箱子
- **目标**：体积利用率 70%~95%，一箱可装多件

### 5.2 中型组件（1500mm ~ 3000mm）
- **代表**：单扇门、窗扇、中小型门板
- **策略**：选择 **C3-C5、C18-C20、C24-C25** 等中型箱子
- **目标**：单件单箱或 2~3 件合箱，利用率 50%~80%

### 5.3 大型组件（3000mm ~ 4500mm）
- **代表**：大尺寸门板、横梁、整扇折叠门
- **策略**：选择 **C2、C11-C12、C26-C27、C30** 等大型/长条箱子
- **目标**：单件单箱，重点保证尺寸能放下，利用率可放宽至 10%~50%

### 5.4 超大型/整扇产品（> 4500mm 或 > 3000mm 双边）
- **代表**：整扇大门窗（如本案例 4572×3048mm）
- **策略**：**必须拆分**
  1. 拆分为窗扇单元
  2. 横梁过长则对半拆分
  3. 型材按根单独包装
  4. 若客户要求整扇运输，需使用 **裸装/框架包装/定制木箱**（超出 C1-C30 范围）

### 5.5 决策流程图
```
产品尺寸
    │
    ├─ 单边 < 1500mm ──→ 小型箱（C1, C10, C15, C17, C21-C23）
    │                      多件合箱，追求高利用率
    │
    ├─ 单边 1500~3000mm ──→ 中型箱（C3-C5, C18-C20, C24-C25）
    │                         单件或少量合箱
    │
    ├─ 单边 3000~4500mm ──→ 大型箱（C2, C11-C12, C26-C27, C30）
    │                         单件单箱，尺寸优先
    │
    └─ 单边 > 4500mm 或双边 > 3000mm
         │
         └─→ 必须拆分 → 按组件分别匹配上述级别
              或 → 定制包装/裸装运输
```

## 六、扩展建议

1. **多件产品装箱**：当前版本为单件产品匹配。如需多件产品装入同一箱，可引入 **3D-Bin-Packing** 算法（如启发式搜索或遗传算法）。
2. **易碎品标记**：可结合 `remark` 字段（如钢化玻璃）自动增大 padding 或降低重量安全系数。
3. **成本优化**：可加入箱子成本数据，在评分中增加成本维度。
4. **历史学习**：记录实际装箱反馈，用机器学习优化评分权重。
